Pipeline di Normalizzazione

Analisi tecnica del processo ETL sviluppato per uniformare dati eterogenei provenienti da MIUR, Eurostat e dataset DNF.

1. Pulizia Avanzata e Standardizzazione

I dataset grezzi presentavano formati inconsistenti per i valori nulli e "rumore" nei dati numerici. Abbiamo implementato una pulizia rigorosa identificando pattern specifici come flag di errore (es. 'b', 'u' di Eurostat) e una vasta gamma di rappresentazioni di valori mancanti.

AnalyzeDNF.py / AnalyzeESTAT.py
# Gestione estesa dei valori nulli
valori_nulli = ['N.A.', 'n.d.', '-', '..', 'n.q.', ':']
df = df.replace(valori_nulli, np.nan)

# Pulizia Regex per dati Eurostat (rimozione flag 'b', 'u')
df[col] = df[col].astype(str).str.replace(r'\s+[bu]', '', regex=True)

# Standardizzazione Colonne
df.columns = df.columns.str.lower().str.strip().str.replace('"', '')

2. Modellazione Dimensionale (Star Schema)

Per garantire analisi efficienti e scalabili, abbiamo trasformato i dati flat in un modello a schema di stella. Le dimensioni (Anno, Regione, Facoltà/Settore) sono state disaccoppiate dalle tabelle dei fatti, creando entità normalizzate con chiavi surrogate generate automaticamente.

  • ✓Generazione automatica di tabelle di lookup
  • ✓Logica incrementale per l'aggiunta di nuovi anni
  • ✓Link tramite Foreign Keys (id_anno, id_regione)
AnalyzeMIUR.py
# Logica incrementale per dimensione Anno
existing_years = set(ANNO_BASE_DF['valore'])
new_years = [y for y in input_years if y not in existing_years]

# Generazione chiavi surrogate
facolta_df['id_facolta'] = range(1, len(facolta_df) + 1)

# Join per Fact Table
df_final = df_pivot.merge(

Trasformazione 1

Splitting Colonne Composite

I dati Eurostat arrivavano con dimensioni aggregate in una singola colonna (es. "A,PC,TOTAL,T,IT"). Abbiamo separato semanticamente questi dati creando colonne distinte per Frequenza, Unità e Geografia.

df[['freq', 'unit', 'geo']] =
df['DIM_COMPOSITA'].str.split(',', expand=True)
Trasformazione 2

Unpivoting (Melt)

Per analizzare serie storiche, abbiamo convertito le strutture "Wide" (anni sulle colonne) in strutture "Long" adatte ai DB relazionali e alla visualizzazione temporale.

pd.melt(df, id_vars=['geo'],
  value_vars=cols_anno,
  var_name='ANNO', value_name='valore')
Trasformazione 3

Pivoting (MIUR)

Al contrario, per i dati MIUR abbiamo pivotato la colonna 'Sesso' per ottenere metriche separate (Iscritti Maschi vs Femmine) sulla stessa riga di osservazione.

df.pivot_table(
  index=['ANNO', 'facolta'],
  columns='sesso', values='iscritti')
